# Dataset Schema

## Table: `user_events`

This table contains event-level user behaviour data from an e-commerce platform.  
Each row represents a single user interaction such as viewing a product page, adding a product to the cart, or completing a purchase.

---

## Table Structure

| Column Name | Data Type | Description |
|-------------|-----------|-------------|
| `event_id` | INTEGER | Unique identifier for each recorded event |
| `user_id` | INTEGER | Unique identifier for the user who performed the event |
| `event_type` | STRING | Type of action performed by the user (e.g., page view, add to cart, purchase) |
| `event_date` | TIMESTAMP | Date and time when the event occurred |
| `product_id` | INTEGER | Identifier of the product associated with the event |
| `amount` | FLOAT | Monetary value of the transaction (only populated for purchase events) |
| `traffic_source` | STRING | Marketing channel that brought the user to the website (e.g., organic, paid ads, social media) |

---

## Funnel Event Types

The sales funnel analysed in this project is defined using the following event types:

| Funnel Stage | Event Type |
|---------------|------------|
| Stage 1 | `page_view` |
| Stage 2 | `add_to_cart` |
| Stage 3 | `checkout_start` |
| Stage 4 | `payment_info` |
| Stage 5 | `purchase` |

---

## Data Characteristics

- Multiple events can be generated by the same user.
- Users may perform the same event multiple times (e.g., multiple page views).
- Funnel metrics are calculated using `COUNT(DISTINCT user_id)` to avoid double counting users within each stage.

---

## Derived Metrics in This Project

The analysis generates the following metrics:

- Funnel stage user counts
- Conversion rates between funnel stages
- Traffic source performance comparison
- Average time from first view to purchase
- Revenue metrics including:
  - Total revenue
  - Average order value (AOV)
  - Revenue per visitor
  - Revenue per buyer

---

## Data Availability

The raw dataset is stored in **Google BigQuery** and is not included in this repository due to potential data licensing restrictions.  

Only SQL queries and analysis results are provided in this project.
